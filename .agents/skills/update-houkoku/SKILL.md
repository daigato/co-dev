# 進捗報告追記スキル (`update-houkoku`)

機能の実装やバグ修正等の作業が完了した際、またはユーザーから「報告して」「houkokuに追記して」「作業内容を記録して」等の指示があった場合、`houkoku.md` に最新の実施内容を分かりやすく追記します。

## 追記ルールと手順

1. **フォーマットと日付**:
   - `houkoku.md` の末尾に追記します。
   - 日付の見出し（例: `🌱7.24`）を記述します。
   - 解決した課題・不具合修正・新規追加機能を分かりやすく構造化（見出し・箇条書き）して記述します。
   - テストや動作確認の結果（例: `-> 対応完了` や `-> 動作確認OK`）を添えます。

2. **記述フォーマット例**:
   ```markdown
   🌱7.24

   ### 実装・修正内容
   1. **[機能名 / 不具合修正名]**
      - 変更・対応した内容の詳細説明。 -> 対応完了
   ```

3. **実行手順**:
   - `houkoku.md` の既存内容を消さずに末尾に追記します。
   - 追記完了後、ユーザーに追記した旨と内容の概要を簡潔に報告します。
